Preguntas

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?
    El internet es una red de redes, los dispositivos interconectados por redes conforman el internet. WWW es un servicio proporcionado por el internet, es la manera de compartir la inforamción
  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    GET: recupera la información identificada por el URI, HEAD: Similar a GET pero solo obtiene la informacón principal (Sin el cuerpo), POST: Solicita al servidor que el recurso del URI tome una nueva entidad, PUT: Solicita que la entidad sea almacenada en el URI, PATCH: Aplica modificaciones parciales a un recurso, DELETE: Solicita que borre el recurso
  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    POST, para que no sea vista ni registrada en ningun lado
  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    GET
  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    Todo esta bien, libre de errrores
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    No, el usuario está intentando acceder a una página que no existe
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    Si, es un error del servidor interno
  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    Elementos que han sido reemplazados en los estándares y no es recomendable usar. Ejemplos: font, center, bgcolor
  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • ¿Qué componentes de estructura y estilo tiene una tabla?
    Filas, columnas, bordes, rellenos, alineación
  • ¿Cuáles son los principales controles de una forma HTML5?
    Inputs de texto, botones, hora, fecha, deslizadores

Investigación 1: Ciclo de vida y de desarrollo de los sistemas de información

Ciclo de vida de los sistemas de información
El ciclo de vida de un sistema de información es un enfoque por etapas por las que pasa cualquier sistema de este tipo. Es importante considerarlo desde el comienzo del desarrollo y seguir cuidadosamente cada una de las fases para garantizar un proyecto exitoso. Aunque al principio puede parecer un proceso lineal, es posible y recomendable regresar a distintas etapas durante el proceso de desarrollo. En el ciclo de vida se pueden identificar principalmente las siguientes etapas: planificación, análisis, diseño, implementación, pruebas, implementación/despliegue y mantenimiento; detalladas a continuación.

  • Planificación – En la etapa de planificación se realizan tareas como delimitar el ámbito del proyecto y el alcance de este, así como posibles riesgos, tiempo y recursos necesarios.
  • Análisis – En la fase de análisis se estudian las necesidades del proyecto, se delimitan los requerimientos.
  • Diseño – En la fase de diseño se establece como se cumplirán los requisitos indicados durante el análisis, se deben describir detalladamente.
  • Implementación – La implementación consiste en integrar el contenido conceptual realizado durante el diseño y llevarlo a una forma física. Es muy importante no comenzar con la implementación si existen dudas o ambigüedades en lo propuesto en el diseño, esto quiere decir que este no fue bien realizado y es necesario tomar más tiempo para dejarlo claro.
  • Pruebas – Esta etapa tiene como objetivo detectar cualquier error cometido durante las etapas anteriores, para así corregirlos, antes de que alguien más tenga que sufrirlos.
  • Despliegue – Es aquí cuando concluyen las etapas de desarrollo y el sistema está listo para funcionar. En este punto es necesario considerar el hardware en el que se utilizará y si se trata de reemplazar algún sistema anterior, hacer la transición sin que el usuario sufra una disrupción.
  • Mantenimiento - El ciclo de vida del sistema no termina una vez que este es implementado, es necesario monitorear su uso para realizar el mantenimiento necesario. Ya sea eliminar defectos que no fueron detectados inicialmente, adaptarlo a nuevas necesidades o implementar nueva funcionalidad.

Es muy importante que llevemos a la práctica este proceso, ya que como hemos estudiado anteriormente, existen muchos proyectos de software que resultan fallidos y seguir una metodología ayuda a reducir la posibilidad de que nuestros proyectos terminen de esa forma.
